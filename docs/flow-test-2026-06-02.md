## 공간파트너(operator) 여정

`로그인 → 온보딩(역할/프로필) → 대시보드 → 공간등록 → 청소요청(3단계) → 매칭대기 → 실시간추적 → 승인 → 리뷰`

**양호:**
- 대시보드: auth·role 가드, 병렬 쿼리, 오늘/이번달 집계
- 청소요청: 3단계 위저드, 공간 없을 때 빈 상태 안내(좋은 UX), 실시간 가격 프리뷰, 고정청소 분기, 베타 안내 명시
- 요청상세: 상태별 액션 게이팅(승인/취소/리뷰/신고), 매칭중 표시, 실시간 위치맵, 취소 환불 미리보기, 단골 등록

**남은 이슈(미수정, 결제 연동 시 처리):**
- 🟠 수동 승인(`handleApprove`)은 job 상태만 APPROVED로 바꾸고 **에스크로 해제(payments HELD→RELEASED) 안 함**. auto-approve는 하는데 수동은 안 함 → 불일치. 결제 연동 때 통일 필요.
- 🟡 대시보드 `monthTotal`이 CANCELED 등 전 상태 합산 → 매출 과대 가능. APPROVED/PAID_OUT만 합산 권장.
- 🟡 청소요청 예약 시 과거 날짜/시간 검증 없음 → 과거 일정 작업 생성 가능.

## 청소파트너(worker) 여정

`로그인 → 온보딩 → clean메인(작업탐색) → 작업상세 → 지원(즉시배정) → 출발(EN_ROUTE) → 도착(ARRIVED, GPS검증) → 청소시작 → 체크리스트+사진 → 완료보고(SUBMITTED) → AI검수 → 승인 → 정산`

**양호:**
- 상태머신(STATUS_FLOW) 명확, 단계별 버튼
- ARRIVED 시 GPS 근접(100m) 검증
- SUBMITTED 전 필수 체크리스트 미완 차단
- 지원 전 정산설정(계좌/세금) 미완료면 차단 + 등록 유도 (좋은 UX)
- 배정 전 주소 마스킹, 배정 후 전체 공개
- 활성작업 중 GPS 45초 핑(호스트 실시간 위치)
- 사진 업로드 → Supabase Storage

**남은 이슈(미수정, 낮음):**
- 🟡 clean 메인 OPEN 작업이 "near me" 주석과 달리 **지역 필터 없이 전국 전체** 노출 (worker 위치 미사용 한계). MVP엔 무방.
- 🟡 상태전이 `advanceStatus`에 status 선조건 가드 없음 (UI·RLS로 보호되나 방어적으로 추가 권장).
- 🟡 ARRIVED GPS 게이트는 위치 거부 시 통과(soft) — 의도된 fallback으로 보임.

## ⏳ 결제 연동 시점 블로커 (이전 리포트 재확인)

테스트 단계라 결제 가짜. 토스 연동할 때 일괄:
1. `/api/payment/confirm` 서버 검증 라우트
2. 결제 우회버튼 제거 + 실제 키
3. success 페이지 confirm 후 Payment(HELD) 생성
4. 수동/자동 승인 모두 에스크로 해제 통일
5. auto-approve service_role + vercel 크론
6. cancel 라우트 실제 토스 환불 연결

## 🔴 새로 발견한 블로커

1. **분쟁 해결 경로 없음** — `admin/disputes`는 목록만, RESOLVED 전환·환불 액션 없음. `DISPUTED` 작업은 cancel/approve 둘 다 불가 → **영구 동결**. 분쟁 해결 API+UI 필요(환불=결제 연동과 함께).
2. **🔴 주민등록번호 평문 저장 (PIPA 위법)** — `profile/tax`가 `resident_id_last`를 `users` 테이블 평문 저장하면서 "암호화 저장" 고지. 한국 개인정보보호법은 주민번호 암호화를 **법적 의무**로 규정. 상용화 전 필수: pgcrypto/Supabase Vault 암호화 또는 미저장 설계. 계좌번호도 동일 권장.
